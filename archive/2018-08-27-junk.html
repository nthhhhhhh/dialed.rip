#0

junk

18/08/27—haskell—generator

Junk is a static site generator written in Haskell for design flexibility. There is no markup, only html where the appropriate id attributes and comments direct the content. There is currently no plans to support rss or any other features. See further for other generators. It is dependent on containers , directory, text and tagsoup.

While I am currently using it, one may want to see other alternatives. Some notable generators are hakyll, hasclunk, blogc, sblg. As far as design flexibility I found hakyll the easiest to get up and running. Though I had a great appreciation for the simplicity of hasclunk. I couldn't find an obvious route to customize sblg extensively but also was impressed with its simplicity.

Currently, the way to use junk is to clone the repo and modify the templates. Take note of the id attributes associated with junk: junkp is populated by blog previews, junkhref links to a blog entry, junka archives a blog post designed in archive_elements.html.
