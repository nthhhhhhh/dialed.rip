nix style local builds

18/08/27—cabal—haskell

A quick few commands to try the new nix-style local builds.


mkdir project
cd project
cabal init
fill out project options
optionally fill out your .cabal file options
cabal new-configure
cabal new-repl
optionally start working
cabal new-build


I have noticed though that running cabal new-run will not run if there have not been any changes, it reports Up to date. I typically use run to execute my binary, not rebuild. Fumbling all the way into the dist directory shouldn't happen until I am finished. Unless I'm missing something, I would appreciate the old behavior that cabal run would execute. For now, I must use cabal new-repl and re-execute there. The above is only meant to get someone started. For more details, as always refer to the documentation..
